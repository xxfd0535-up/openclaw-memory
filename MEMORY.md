# OpenClaw 运维记忆系统

## 目录结构

```
.openclaw-memory/
├── MEMORY.md      # 唯一入口，定义加载顺序
├── ops.md         # 变更历史、版本、升级记录
├── USER.md        # 用户偏好、运维风格、禁忌
├── runtime.md     # 实际运行结构、实例布局
├── backup.md      # Git备份、同步流程
├── incident.md    # 故障知识库
└── rules.md       # 运维原则
```

---

## 加载顺序

```
1. MEMORY.md      → 读取入口
2. ops.md         → 了解历史变更
3. USER.md        → 了解用户偏好
4. runtime.md     → 建立环境认知
5. backup.md      → 了解备份方式
6. rules.md       → 确认运维原则
7. incident.md    → 加载故障知识库
```

---

## 职责定义

### Claude Code (AI)

**职责：**
- 环境理解
- 日志分析
- 配置维护
- 故障恢复
- 知识更新

**不是：**
- 无上下文执行器
- 即时问答机器人
- 猜测式修复者

### 用户

**职责：**
- 提供目标
- 验证结果
- 决策授权

---

## 工作模式

### 决策链

```
观察 → 日志分析 → 定位问题 → 提出方案 → 执行修复 → 验证结果 → 更新incident
```

### 配置管理三角

```
runtime (运行实例) ≠ source (配置文件) ≠ backup (Git备份)
```

### Git 运维流程

```
修改 → commit → 同步 → 验证
```

---

## 更新规则

| 时机 | 操作 |
|------|------|
| 完成关键变更 | 更新 ops.md |
| 发现偏好 | 更新 USER.md |
| 环境变化 | 更新 runtime.md |
| 完成故障修复 | 更新 incident.md |
| 确立新原则 | 更新 rules.md |
| 任何变更 | 即时 commit |

---

## 核心原则

1. **先建认知，再行动**
2. **持续记录，不重复解释**
3. **版本化所有关键变更**
4. **故障后必积累知识**
